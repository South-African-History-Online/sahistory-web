01

Front page / Catalogue home

Desktop · container-standard 1200
SiteHeader · wordmark + nav + search icon
SearchField · front-door query (full width)
ArchiveStatusBar · record counts (mono)
Current feature · EDITORIAL register (Archivo)
This day in history · chronology entry
Browse the archive · ruled type index (6 content types)
IndexTable · recently added records

Purpose

Not a splash. The catalogue front page: query box, live record counts, one editorial feature, the type index, and the most recent records as a table.

DS components

SiteHeader, SearchField, ArchiveStatusBar, HomeFeature (editorial), Timeline teaser, IndexTable.

Drupal mapping

  • Header/footer = theme regions + page-footer SDC.
  • Feature = a curated Node queue block.
  • Status counts = a small cached Views aggregate block.
  • Recent records = a Views table (fields: ref, type, title, dates, status).
02

Content-type landing (Biographies / Topics / Places / Events / Archive)

Desktop · one parameterized template
Landing header · type tab + title + count + intro
Filter bar · era / theme / place Tags + sort
IndexTable OR ArchiveCard grid (toggle)
Facets · A–Z / era / place
Featured record

Purpose

One template drives all five content-type indexes. The type only changes the accent, the tab label and the default facets.

DS components

RecordHeader (landing variant), Tag filters, IndexTable / ArchiveCard grid toggle, facet list.

Drupal mapping

  • A single Views display with a contextual filter on content type.
  • Exposed filters = the Tag/facet UI (Facets module).
  • Grid/table = two Views formats behind a toggle.

05

Search & browse · the front door

Desktop · query → index table
SearchField (large) + scope chips
Facets · type · era · place (checkbox)
IndexTable · results (ref · type · record · dates · src)

Purpose

Fast, forgiving, prominent. A query returns a ruled index, not cards. Facets refine; results stay legible and dense.

DS components

SearchField, facet checklist, IndexTable (sortable headers), pagination.

Drupal mapping

  • Search API + Views; Facets module for the left rail.
  • Forgiving search = fuzzy/typo tolerance in Search API.
  • Type column swatch from content type.

Purpose

Mobile is the majority case on mid-range Android over metered data. Tables collapse to stacked rows; rails move below content; facets become a bottom sheet. Hit targets ≥ 44px.

Performance

  • Self-hosted, subset fonts; system-font fallback while swapping.
  • Progressive image loading; duotone via CSS, not baked assets.
  • No table horizontal-scroll · re-layout to cards.

Drupal mapping

  • Single responsive theme; Views row templates switch format by breakpoint via CSS.
